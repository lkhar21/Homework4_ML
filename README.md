# Homework 4 – Machine Learning

## Sanity Check

სრულმასშტაბიანი ტრენინგის დაწყებამდე ჩატარდა ლექციაზე განხილული Forward და Backward პასების სისწორის შემოწმება. ამ მიზნით მონაცემთა ბაზიდან ავიღეთ მხოლოდ ერთი ბეტჩი (Batch Size = 64) და ვცადეთ მოდელის (MiniCNN) მაქსიმალური გადაწვრთნა (Overfitting on One Batch).

### შედეგი

50-ე ეპოქისთვის მოდელმა მიაღწია 100.00%-იან სიზუსტეს (Accuracy), ხოლო დანაკარგის ფუნქციის (Loss) მნიშვნელობა პრაქტიკულად ნულს გაუტოლდა.

### დასკვნა

ექსპერიმენტმა დაადასტურა, რომ კოდში ლოგიკური შეცდომები არ არსებობს, გრადიენტები სწორად ვრცელდება ქსელში და მოდელს მონაცემების სწავლა ეფექტურად შეუძლია.

# მოდელის ეტაპობრივი განვითარება და ექსპერიმენტების ანალიზი

მოდელის გაუმჯობესება განხორციელდა ეტაპობრივად — მარტივი არქიტექტურიდან უფრო ღრმა და ოპტიმიზებულ ქსელამდე. თითოეული ექსპერიმენტი დამოუკიდებლად ილოგებოდა Weights & Biases (WandB) პლატფორმაზე, რაც საშუალებას გვაძლევდა დეტალურად გვედევნებინა ტრენინგის პროცესი და სხვადასხვა მიდგომის ეფექტიანობა შეგვედარებინა.

| ------------------------------------------- | ----------------------------------- | ----------------------------- | ----------- | --------------------------------------------------- |

# არქიტექტურა 1 – Architecture_1_Baseline

### მიღებული გადაწყვეტილება

საწყის ეტაპზე გამოყენებული იყო მარტივი კონვოლუციური ნეირონული ქსელი, რომელიც შედგებოდა ორი კონვოლუციური ფენისგან (32 და 64 ფილტრი). მოდელში არ იყო გამოყენებული რეგულარიზაციის მეთოდები და ტრენინგი მიმდინარეობდა SGD ოპტიმიზატორით, Learning Rate = 0.01.

### შედეგების ანალიზი

მოდელმა მხოლოდ 10 ეპოქის განმავლობაში ტრენინგის სიზუსტე 82.37%-მდე გაზარდა, თუმცა ვალიდაციის სიზუსტემ მხოლოდ 12.20% შეადგინა. ტრენინგის Loss სტაბილურად მცირდებოდა (0.50-მდე), მაშინ როდესაც Validation Loss მნიშვნელოვნად იზრდებოდა (4.77-მდე).

### დასკვნა

მიღებული შედეგი წარმოადგენს კლასიკურ Overfitting-ს. მოდელმა დაიმახსოვრა სასწავლო მონაცემების კონკრეტული ნიმუშები, თუმცა ვერ შეძლო ახალი მონაცემების გენერალიზაცია.

---

# არქიტექტურა 2 – Architecture_2_Baseline_With_Dropout

### მიღებული გადაწყვეტილება

Overfitting-ის შესამცირებლად მოდელში დაემატა Dropout ფენები:

* Dropout = 0.25 კონვოლუციური ფენების შემდეგ;
* Dropout = 0.50 Fully Connected ფენის წინ.

ასევე ოპტიმიზატორი შეიცვალა Adam-ით და Learning Rate შემცირდა 0.001-მდე.

### შედეგების ანალიზი

Dropout-მა მნიშვნელოვნად შეამცირა გადაწვრთნის ეფექტი. ტრენინგისა და ვალიდაციის მეტრიკები ერთმანეთთან უფრო დაახლოვდა, რაც მიუთითებდა უკეთეს გენერალიზაციაზე.

თუმცა, ქსელის შედარებით მცირე ზომის გამო, მოდელს კვლავ უჭირდა რთული მახასიათებლების შესწავლა.

### დასკვნა

მიუხედავად იმისა, რომ Overfitting შემცირდა, მოდელის სიმძლავრე არასაკმარისი აღმოჩნდა. გამოვლინდა Underfitting-ის ნიშნები, რაც მიუთითებდა უფრო ღრმა არქიტექტურის საჭიროებაზე.

---

# არქიტექტურა 3A – Architecture_3A_Optimized_With_Augmentation

### მიღებული გადაწყვეტილება

მოდელის შესაძლებლობების გასაზრდელად დაემატა დამატებითი კონვოლუციური ფენები და 128-ფილტრიანი ბლოკები. ასევე გამოყენებული იქნა:

* Batch Normalization;
* Dropout;
* მონაცემთა აუგმენტაცია:

  * RandomHorizontalFlip;
  * RandomRotation.

### შედეგების ანალიზი

აუგმენტაციის გამოყენებამ მნიშვნელოვნად გააუმჯობესა მოდელის გენერალიზაციის უნარი. ყოველ ეპოქაზე მოდელი იღებდა ოდნავ მოდიფიცირებულ სურათებს, რის შედეგადაც ის სწავლობდა არა კონკრეტულ პიქსელურ განლაგებებს, არამედ სახის მნიშვნელოვან მახასიათებლებს — თვალების, წარბების, ტუჩებისა და სხვა გეომეტრიული სტრუქტურების ფორმებს.

Validation Loss გახდა უფრო სტაბილური და მნიშვნელოვნად შემცირდა განსხვავება ტრენინგისა და ვალიდაციის შედეგებს შორის.

### დასკვნა

გაუმჯობესებულმა არქიტექტურამ შეძლო მონაცემთა უკეთესი გენერალიზაცია და მნიშვნელოვნად გადაჭრა წინა მოდელებში გამოვლენილი პრობლემები.

---

# არქიტექტურა 3B – Architecture_3B_Optimized_AdamW_LowLR

### მიღებული გადაწყვეტილება

საბოლოო მოდელის მისაღებად განხორციელდა ჰიპერპარამეტრების შერჩევა (Hyperparameter Tuning).

ცვლილებები მოიცავდა:

* Adam ოპტიმიზატორის ჩანაცვლებას AdamW-ით;
* Learning Rate-ის შემცირებას 0.0001-მდე;
* Weight Decay მექანიზმის გამოყენებას უკეთესი რეგულარიზაციისთვის.

### შედეგების ანალიზი

AdamW ოპტიმიზატორმა და დაბალმა Learning Rate-მა უზრუნველყო უფრო სტაბილური და ზუსტი სწავლება. მოდელი თანმიმდევრულად ამცირებდა Loss-ს და უკეთ აღწევდა ფუნქციის გლობალურ მინიმუმს.

ტრენინგისა და ვალიდაციის მეტრიკებს შორის განსხვავება მინიმალური გახდა, რაც მიუთითებს გენერალიზაციის მაღალ ხარისხზე.

### დასკვნა

Architecture_3B გახდა პროექტის საუკეთესო მოდელი. მან უზრუნველყო ოპტიმალური ბალანსი Bias-სა და Variance-ს შორის და აჩვენა ყველაზე მაღალი სიზუსტე ვალიდაციის მონაცემებზე.
